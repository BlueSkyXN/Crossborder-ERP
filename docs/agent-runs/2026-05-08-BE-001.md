# BE-001 初始化后端项目

## 输入

- 任务文件：`docs/ai-dev-baseline/agent-execution/tasks/BE-001.md`
- 关联规格：`03-technical-architecture.md`、`04-api-database-contract.md`、`14-module-implementation-spec.md`
- 人工约束：no-Docker；SQLite 为当前唯一真实验证数据库；PostgreSQL/MySQL/Redis/Celery 只保留配置兼容边界，不做真实验证。

## Agent 决策

- 使用 `uv` 和项目本地 `.venv` 管理 Python 3.12 环境，不写入全局 Python。
- 拆分 Django settings 为 `base/local/test/production`，默认本地和测试均走 SQLite。
- 以 `apps.common` 承载统一响应、异常处理、分页和 health API，避免后续业务 app 重复实现基础协议。
- 增加 GitHub Actions 后端检查，当前只运行 no-Docker 后端检查和 pytest。

## 修改

- 初始化 Django/DRF/drf-spectacular/pytest 后端基线。
- 新增 `/api/v1/health`、`/api/v1/schema/`、`/api/v1/docs/`。
- 新增统一响应、错误响应、异常处理和分页类。
- 新增后端 CI workflow。

## 验证

```bash
uv sync --locked --dev
uv run python manage.py check
uv run python manage.py spectacular --file /tmp/crossborder-openapi.yaml
uv run pytest
actionlint .github/workflows/ci.yml
```

结果：Django check 通过；OpenAPI schema 可生成；pytest `3 passed`；workflow 通过 `actionlint`。

## 未验证边界

- PostgreSQL/MySQL/Redis/Celery 均为 `configured_unverified`，本任务没有启动或真实连接这些服务。
