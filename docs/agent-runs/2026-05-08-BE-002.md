# BE-002 后台 IAM 和 RBAC

## 输入

- 任务文件：`docs/ai-dev-baseline/agent-execution/tasks/BE-002.md`
- 关联规格：`02-domain-model-state-machines.md`、`04-api-database-contract.md`、`14-module-implementation-spec.md`
- 人工约束：后台管理员与普通会员用户分离；no-Docker；SQLite 为当前唯一真实验证数据库。

## Agent 决策

- 新增独立 `apps.iam`，实现 `AdminUser`、`Role`、`Permission`，不复用后续会员用户模型。
- 后台 token 使用 JWT，但增加 `token_scope=admin`，后台接口只接受 admin scope。
- RBAC 先覆盖菜单权限和角色列表权限，满足 P1 骨架和后续后台菜单接入。
- `seed_demo` 先创建超级管理员、仓库人员、财务人员、采购人员，后续任务可继续扩展该命令。

## 修改

- 新增 IAM models、migration、serializers、views、permissions、authentication、services 和 tests。
- 新增后台接口：`POST /api/v1/admin/auth/login`、`GET /api/v1/admin/me`、`GET /api/v1/admin/menus`、`GET /api/v1/admin/roles`。
- 为自定义后台 Bearer token 增加 OpenAPI authentication scheme。

## 验证

```bash
uv run python manage.py check
uv run python manage.py spectacular --file /tmp/crossborder-openapi-be002.yaml
uv run pytest apps/iam apps/common
uv run python manage.py migrate --noinput
uv run python manage.py seed_demo
```

结果：Django check 通过；OpenAPI schema 可生成；pytest `9 passed`；SQLite migration 和 `seed_demo` 执行成功。

## 未验证边界

- PostgreSQL/MySQL/Redis/Celery 仍为 `configured_unverified`。
- 当前只验证后台 IAM/RBAC 骨架，未实现普通会员登录和用户数据隔离。
