# BE-003 用户和会员资料

## 输入

- 任务文件：`docs/ai-dev-baseline/agent-execution/tasks/BE-003.md`
- 关联规格：`13-integrated-product-spec.md`、`14-module-implementation-spec.md`、`04-api-database-contract.md`
- 人工约束：不实现验证码和找回密码；用户 token 不得访问后台接口；SQLite 为当前唯一真实验证数据库。

## Agent 决策

- 新增独立 `apps.members`，实现普通 `User` 和 `MemberProfile`，不复用后台 `AdminUser`。
- 用户 token 使用 JWT 且带 `token_scope=member`，与后台 `token_scope=admin` 分离。
- 会员收件标识先按用户 ID 生成 `member_no=M000001` 和 `warehouse_code=CB000001`，后续仓库模块可继续组合专属仓库地址。
- `seed_demo` 扩展为同时创建 IAM 和测试会员数据。

## 修改

- 新增用户注册、登录、登出、当前用户、资料更新接口。
- 新增 members models、migration、authentication、schema、serializers、services、views 和 tests。
- 更新任务状态到 `BE-004`。

## 验证

```bash
uv run python manage.py check
uv run python manage.py spectacular --file /tmp/crossborder-openapi-be003.yaml
uv run pytest apps/members apps/iam apps/common
uv run python manage.py migrate --noinput
uv run python manage.py seed_demo
```

结果：Django check 通过；OpenAPI schema 可生成且无 warning/error；pytest `16 passed`；SQLite migration 和 `seed_demo` 执行成功。

- 验证码、找回密码、复杂会员等级权益保持 `TODO_CONFIRM` 或后续任务处理。
