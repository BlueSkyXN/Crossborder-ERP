# BE-005 包裹预报和扫描入库

输入来自 `docs/ai-dev-baseline/agent-execution/tasks/BE-005.md`，基于 `BE-003` 会员体系、`BE-004` 仓库配置和当前无 Docker/无外部数据库约束，实现 Parcel/WMS 后端最小闭环。本轮未引入 PostgreSQL/MySQL、Redis/Celery 或系统级服务，只在 Django 项目内新增 `apps.parcels` 并用 SQLite 验证迁移。

Agent 在 `ai/be005-parcel-wms` 分支新增 `Parcel`、`ParcelItem`、`ParcelPhoto`、`InboundRecord`、`UnclaimedParcel` 模型，注册 `parcels` app 和 API 路由。服务层实现 `forecast_parcel`、`scan_inbound`、`inbound_parcel`、`claim_unclaimed_parcel`；用户端 API 覆盖预报、列表、详情、可打包列表；后台 API 覆盖包裹列表、扫描入库、按包裹入库、无主包裹列表/登记。状态流转限制为 `PENDING_INBOUND -> IN_STOCK`，非法重复入库返回 `STATE_CONFLICT`。

验证结果：已执行 `uv run pytest apps/parcels`、`uv run pytest`、`uv run python manage.py check`、`uv run python manage.py spectacular --file /tmp/crossborder-openapi-be005.yaml`、`uv run python manage.py makemigrations --check --dry-run`、`uv run python manage.py migrate`，均通过。测试覆盖预报创建待入库、后台入库转在库并生成 `InboundRecord`、重复入库冲突、非本人不可访问、未知单号扫描生成无主包裹、匹配单号扫描入库、无权限后台用户拒绝。
