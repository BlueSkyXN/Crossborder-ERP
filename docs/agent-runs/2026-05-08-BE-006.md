# 2026-05-08 BE-006

输入来自 `docs/ai-dev-baseline/agent-execution/tasks/BE-006.md`，目标是实现 Waybill 从用户申请打包、后台审核、设置费用到待付款的后端基础。继续遵守本阶段约束：不使用 Docker，不安装或启动 PostgreSQL/MySQL/Redis，SQLite first，状态流转不在 view 层直接修改。

本轮新增 `apps.waybills`，包含 `Waybill`、`WaybillParcel`、状态枚举、迁移、serializer、service 和 API。用户端接口覆盖 `POST /api/v1/waybills`、`GET /api/v1/waybills`、`GET /api/v1/waybills/{id}`；后台接口覆盖 `GET /api/v1/admin/waybills`、`POST /api/v1/admin/waybills/{id}/review`、`POST /api/v1/admin/waybills/{id}/set-fee`。`create_waybill` 只允许当前会员选择同仓且 `IN_STOCK` 的 Parcel，并在同一事务中把 Parcel 推进为 `PACKING_REQUESTED`；`review_waybill` 推进到 `PENDING_PACKING`；`set_waybill_fee` 推进到 `PENDING_PAYMENT`；`cancel_waybill` 服务保留取消并释放包裹能力。

验证已完成：`cd backend && uv run pytest apps/waybills`、`cd backend && uv run pytest`、`cd backend && uv run python manage.py check`、`cd backend && uv run python manage.py makemigrations --check --dry-run`、`cd backend && uv run python manage.py spectacular --file /tmp/crossborder-be006-openapi.yaml --validate`、`actionlint .github/workflows/ci.yml` 均通过。测试覆盖创建运单、包裹状态变更、后台审核、设置费用、非法状态流转返回 `STATE_CONFLICT`、后台权限拒绝和取消释放包裹。
