# 2026-05-08 BE-007

输入来自 `docs/ai-dev-baseline/agent-execution/tasks/BE-007.md`，目标是实现钱包、支付单、钱包流水、后台充值/扣减和用户余额支付。继续遵守本阶段约束：不接真实在线支付回调，不用 float，SQLite first，不启动 Docker、PostgreSQL/MySQL/Redis。

本轮新增 `apps.finance`，包含 `Wallet`、`WalletTransaction`、`PaymentOrder`、`RechargeRequest`、状态/类型枚举、迁移、serializer、service 和 API。用户端接口覆盖 `GET /api/v1/wallet`、`GET /api/v1/wallet/transactions`、`POST /api/v1/waybills/{id}/pay`；后台接口覆盖 `GET /api/v1/admin/wallet-transactions`、`POST /api/v1/admin/users/{id}/wallet/recharge`、`POST /api/v1/admin/users/{id}/wallet/deduct`、`GET /api/v1/admin/payment-orders`。`pay_with_wallet` 在同一事务中锁定 Waybill 与 Wallet，创建/复用 PaymentOrder，扣减余额、写入流水，并把 Waybill 推进到 `PENDING_SHIPMENT`；重复支付通过 `idempotency_key` 返回既有支付结果，不重复扣款。

验证已完成：`cd backend && uv run pytest apps/finance`、`cd backend && uv run pytest`、`cd backend && uv run python manage.py check`、`cd backend && uv run python manage.py makemigrations --check --dry-run`、`cd backend && uv run python manage.py spectacular --file /tmp/crossborder-be007-openapi.yaml --validate`、`actionlint .github/workflows/ci.yml` 均通过。测试覆盖后台充值生成流水、后台扣减、余额不足返回 `INSUFFICIENT_BALANCE`、余额支付扣款并推进 Waybill、重复支付不重复扣款、财务权限拒绝和后台支付单列表。
