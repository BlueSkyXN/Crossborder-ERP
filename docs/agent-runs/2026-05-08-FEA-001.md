# FEA-001 后台登录和 Layout

输入来自 `docs/ai-dev-baseline/agent-execution/tasks/FEA-001.md`，并沿用本项目隔离约束：前端依赖仅安装在仓库本地 `node_modules`，不安装或启动外部数据库、Redis、Docker。Agent 在 `ai/fea001-admin-shell` 分支初始化 `admin-web`，采用 React、Vite、TypeScript、Ant Design、React Router、TanStack Query、Axios、Zustand，完成 API client、统一 `code/message/data` 解包、admin JWT 保存、退出登录、未登录跳转、登录页、后台 Layout、动态菜单、403/404 和基础工作区。

关键取舍：API base URL 集中在 `src/api/client.ts` 和 Vite proxy，不在页面散落拼接；菜单优先读取 `GET /api/v1/admin/menus`，失败或空数据时使用种子权限兜底；视觉不复刻旧系统，采用低装饰、高密度的后台工作台风格。CI 已补充 frontend job，使用 Node 24 与 `pnpm@10.30.3` 执行 admin-web lint/build。

验证结果：已执行 `pnpm install`、`pnpm --filter @crossborder-erp/admin-web lint`、`pnpm --filter @crossborder-erp/admin-web build`、`pnpm lint`、`pnpm build`、`actionlint .github/workflows/ci.yml`。浏览器验收覆盖：未登录访问 `/warehouses` 跳转登录；admin 登录后回到 `/warehouses` 并显示菜单/Layout；buyer 登录访问 `/warehouses` 显示 403；移动端 390px 视口使用 Drawer 菜单且无明显遮挡。
