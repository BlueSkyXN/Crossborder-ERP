# FEM-001A 移动 H5 登录、底部导航、寄件首页

输入来自 `docs/ai-dev-baseline/agent-execution/tasks/FEM-001A.md`，基于 `BE-003` 会员登录与 `BE-004` 仓库地址接口实现移动 H5 基础入口。本轮仍不引入 Docker、PostgreSQL/MySQL、Redis/Celery 或系统级依赖，只在 `mobile-h5` workspace 内增加本地前端依赖并复用 SQLite 后端做联调。

Agent 在 `ai/fem001a-mobile-h5-home` 分支初始化 `mobile-h5`，采用 React + Vite + TypeScript + Ant Design Mobile + React Router + TanStack Query + Axios + Zustand。页面覆盖会员登录、受保护路由、底部五栏、寄件首页、集运/直邮 Tab、仓库 Tab、地址详情、一键复制、发布预报/申请打包入口占位、loading/empty/error 状态。联调时发现移动端必须按后端真实契约读取 `/warehouses` 的 `{ items }`，已修正。

验证结果：已执行 `pnpm --filter @crossborder-erp/mobile-h5 lint`、`pnpm --filter @crossborder-erp/mobile-h5 build` 并通过。浏览器验收覆盖：390px 移动视口未登录跳转登录页；使用 `user@example.com / password123` 登录后显示 `测试用户`、会员号 `M000001`、仓库标识 `CB000001`、底部五栏、集运/直邮与仓库 Tab；寄件页可展示 `集运仓 CB000001` 完整地址，复制按钮点击后出现 `已复制` 状态；浏览器 console 无 error。已知边界：`antd-mobile` 的上游 `@react-spring` peer 仍提示未覆盖 React 19，但当前页面避开 imperative 弹层路径，lint/build 和浏览器验收通过。
