# FEU-001A 用户 Web 登录、控制台、仓库地址

输入来自 `docs/ai-dev-baseline/agent-execution/tasks/FEU-001A.md`，基于 `BE-003` 会员登录和 `BE-004` 仓库地址接口实现用户 Web 基础入口。本轮未引入 Docker、PostgreSQL/MySQL/Redis 或系统级依赖，只使用项目内 `pnpm` workspace、后端本地 `.venv` 和 SQLite 做联调验证。

Agent 在 `ai/feu001a-user-web-dashboard` 分支初始化并完善 `user-web`，采用 React + Vite + TypeScript + React Router + TanStack Query + Axios + Zustand + CSS Modules。页面覆盖会员登录、持久化登录态、未登录 redirect、会员资料、余额和业务状态占位、专属仓库地址卡、一键复制、loading、empty、error。CI 前端任务从仅验证 `admin-web` 扩展为执行 workspace 级 `pnpm lint` 与 `pnpm build`，从而覆盖 `admin-web` 和 `user-web`。

验证结果：已执行 `pnpm lint`、`pnpm build`、`actionlint .github/workflows/ci.yml`，均通过。浏览器验收覆盖：访问 `/dashboard` 未登录会跳转登录页；使用 `user@example.com / password123` 登录后可看到 `测试用户`、会员号 `M000001`、仓库标识 `CB000001` 和集运仓地址；点击复制按钮显示 `已复制`。桌面和 390px 移动视口未发现明显遮挡。
