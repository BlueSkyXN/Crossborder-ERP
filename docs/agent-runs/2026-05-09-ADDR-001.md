# 2026-05-09 ADDR-001

## 任务

补齐地址簿与运单地址复用能力：后端 address API，User Web/Mobile H5 地址管理入口，以及运单创建时通过 `address_id` 生成收件快照。

## 交付

- 新增 `backend/apps/addresses`，支持会员地址列表、新增、编辑、停用、设默认，并生成 SQLite migration。
- 运单创建支持 `address_id`，后端会读取当前会员启用地址生成 `recipient_snapshot`；手工填写收件信息的 fallback 继续保留。
- User Web 新增 `/addresses` 地址簿页面，并在控制台和运单申请表接入地址选择。
- Mobile H5 新增 `/me/addresses` 地址簿页面，并在“我的”和申请打包页接入地址选择。
- API E2E 增加“创建地址 -> 运单使用地址 -> 修改地址后历史运单 snapshot 不漂移”断言。
