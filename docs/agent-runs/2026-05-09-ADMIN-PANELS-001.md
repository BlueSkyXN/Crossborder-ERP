# ADMIN-PANELS-001 后台占位面板真实化

## 背景

完成态审计继续检查真实代码后发现，Admin Web 的 `/dashboard` 和 `/roles` 仍由通用 `WorkspacePage` 承载，页面内存在固定统计和固定工作列表。这会削弱“后台面板已真实整合，而不是 demo”的证明力度。

## 关键实现

- 后端新增 `GET /api/v1/admin/dashboard`，复用 Admin token 和 `dashboard.view` 权限，按当前管理员角色裁剪可见模块。
- Dashboard 聚合真实模型计数：会员、仓库配置、包裹/无主认领、运单/发货批次、财务、代购、商品、客服、内容和审计。
- Admin Web `/dashboard` 新增真实运营控制台，展示摘要卡、工作队列、模块健康概览和最近审计动作。
- Admin Web `/roles` 新增真实角色权限页，展示角色列表、权限展开项和菜单权限覆盖矩阵。
- Browser Smoke 增加后台 dashboard、roles、warehouses、purchases、products、tickets、content 面板断言。

## 验证

- `cd backend && uv run pytest apps/iam/tests/test_admin_auth.py -q`：9 passed。
- `pnpm --filter admin-web lint`：passed。
- `pnpm --filter admin-web build`：passed。
- 其余全量验证以本轮最终命令和 CI 为准。

## 未验证边界

- 本轮只做只读 dashboard 和角色权限展示，不新增角色编辑、权限分配、create/update/delete 细粒度 RBAC。
- PostgreSQL/MySQL/Redis/Celery/Docker 不真实验证。
- 真实监控告警、外部审计归档和 staging 运维面板仍属后续生产化任务。
