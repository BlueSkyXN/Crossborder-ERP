# AUDITLOG-001 操作审计日志

## 输入

- 用户要求继续补齐剩余任务，目标包括满足原始 ChatGPT/Gemini 两套分析报告、前后端数据库设计完整、文档和 CI 齐全、SQLite + demo 数据优先、避免污染本地环境。
- `docs/source-report-gap-map.md` 和 `docs/ai-dev-baseline/04-api-database-contract.md` 均指出后台操作日志仍缺。

## 关键实现

- 新增 `backend/apps/audit`，包含 `AuditLog`、`AuditOperatorType`、serializer、Admin API、middleware 和 migration。
- 后台 `/api/v1/admin/**` 的 `POST/PUT/PATCH/DELETE` 请求自动写入审计日志，记录操作人、动作、对象、请求路径、状态码、IP、UA、脱敏请求数据和脱敏响应数据。
- 财务高风险服务层补充显式审计，覆盖应付款创建/更新/确认/核销/取消、线下汇款审核/取消、后台钱包人工加扣款。
- IAM seed 增加 `audit.logs.view` 菜单权限；Admin Web 增加 `/audit-logs` 查询页。
- API E2E 补审计日志断言；Browser Smoke 增加 Admin Web 审计日志入口检查。

## 边界

- 不接外部 SIEM、长期归档、保留期策略、审计告警或合规报表。
- 不拆分 RBAC create/update/delete 细粒度权限。
- 不新增外部依赖，不启动 Docker、PostgreSQL/MySQL/Redis/Celery。
