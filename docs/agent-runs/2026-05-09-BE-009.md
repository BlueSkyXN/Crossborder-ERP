# 2026-05-09 BE-009

## 任务

最小代购和到货转 Parcel 后端基础。

## 交付

- 新增 `products`、`purchases` Django apps，落地商品分类、商品、SKU、购物车、代购订单、代购明细和采购任务模型及 migration。
- 新增商品列表/详情 API，购物车新增、列表、更新和删除 API。
- 新增用户代购单 API，覆盖购物车下单、手工代购、订单列表/详情和余额支付。
- 新增后台代购订单 API，覆盖订单列表/详情、审核、采购、标记到货和到货转 `Parcel.IN_STOCK`。
- 代购余额支付复用现有 `PaymentOrder`、`Wallet`、`WalletTransaction`，使用 `PURCHASE_ORDER` 和 `PURCHASE_PAYMENT` 类型。
- 到货转包裹会创建 `Parcel`、`ParcelItem` 和 `InboundRecord`，并回填 `PurchaseOrder.converted_parcel`，转入包裹可继续创建 Waybill。
- `seed_demo` 新增商品和 SKU demo 数据。

## 边界

- 不接淘宝、1688、拼多多真实接口。
- 不做自动采购、自动外链解析、真实采购凭证上传。
- PostgreSQL/MySQL/Redis/Docker 仍不在本轮真实验证范围内。
- 转包裹接口当前要求后台显式提供 `warehouse_id` 和 `weight_kg`，避免在订单模型没有仓库字段时默认脑补仓库。
