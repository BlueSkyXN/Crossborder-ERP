# CI-EVIDENCE-001 Agent 证据门禁

## 输入

- 用户要求每轮 Agent 任务记录有固定落点，并可作为证明材料提交进 git。
- 现有 `docs/agent-runs/` 已有任务摘要，但 CI 只验证代码、前端和浏览器 smoke，未自动检查任务图和证明材料一致性。
- 任务卡：`docs/ai-dev-baseline/agent-execution/tasks/CI-EVIDENCE-001.md`。

## Agent 决策

- 新增标准库 Python 脚本，避免为了证明材料门禁引入新的依赖。
- CI 独立增加 `Agent Evidence` job，在后端和前端依赖安装前即可运行。
- 门禁只检查证据结构、任务图/current-state 一致性、Agent run 基础完整性和未验证边界表述；不把它包装成业务正确性证明。

## 修改

- 新增 `scripts/ci/validate_agent_evidence.py`。
- 新增 `npm run evidence`。
- `.github/workflows/ci.yml` 新增 `Agent Evidence` job。
- 补齐少量既有 Agent run 记录中的验证结果表述，避免证明材料停留在命令名或空泛记录。

## 验证

- `python3 scripts/ci/validate_agent_evidence.py`：通过，检查 51 个任务和 50 份 Agent run 摘要。
- `npm run evidence`：通过。
- `actionlint .github/workflows/ci.yml`：通过。
- `git diff --check`：通过。
- YAML parse：`yaml ok`，仅出现外置盘 `/Volumes/TP4000PRO` world-writable PATH warning。

## 未验证边界

- 该门禁不替代 `pytest`、`npm run e2e`、`npm run e2e:browser`、`pnpm lint`、`pnpm build` 或人工/Agent 代码审查。
- PostgreSQL/MySQL/Redis/Celery/Docker 仍不真实验证。
