# CONTENT-001 内容 CMS

## 输入

- 任务文件：`docs/ai-dev-baseline/agent-execution/tasks/CONTENT-001.md`
- 关联规格：`docs/source-report-gap-map.md`、`docs/production-readiness-backlog.md`
- 人工约束：no-Docker、SQLite-first；PostgreSQL/MySQL/Redis/Celery 不真实验证。

## Agent 决策

- 内容正文先使用 plain text/Markdown-like 文本，不接外部富文本编辑器或上传能力。
- 后台内容管理使用 `content.view` 权限进入；本轮不拆 create/update/delete 细粒度权限。
- 用户端公开 API 只返回 `PUBLISHED` 内容，`DRAFT`/`HIDDEN` 不可读取。
- 条款、隐私政策和帮助正文先放演示占位，正式文案仍需业务/法务确认。

## 修改

- 后端新增 `content` app，包含内容分类、内容条目、发布/隐藏服务、公开只读 API 和后台 CRUD API。
- `seed_demo` 增加公告、帮助、条款、隐私政策和关于我们 demo 内容。
- Admin Web 新增 `/content` 内容管理页，支持列表、筛选、创建、编辑、发布、隐藏和分类停用。
- User Web 新增 `/content` 内容中心，并从 dashboard 增加“帮助公告”入口。
- Mobile H5 新增 `/me/content` 帮助公告页，并从“我的/Support”进入。
- `npm run e2e` 增加 `[CONTENT-001] 内容 CMS 链路完成`，覆盖草稿不可读、发布可读、隐藏后不可读。
- 更新任务状态、task graph、backlog、gap map、README、部署文档和已知边界，并创建下一任务 `IMPORT-001`。
