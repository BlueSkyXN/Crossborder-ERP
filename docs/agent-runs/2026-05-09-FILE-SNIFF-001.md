# FILE-SNIFF-001 文件上传内容签名校验

## 输入

- 完成度审计显示整体目标仍不能宣布完整生产级完成，文件安全仍在对象存储、病毒扫描、缩略图等生产化边界中。
- 当前用户约束仍是 SQLite-first、no-Docker、不安装全局依赖、不下载额外浏览器或服务。

## Agent 决策

- 选择不依赖外部服务、可本地验证的文件上传安全增强，先补基础内容签名校验。
- 继续保留本地 `MEDIA_ROOT` 和现有 `StoredFile.checksum_sha256` 设计，不新增数据库字段或 migration。
- 按文件扩展名做 MIME 一致性校验，按文件头识别 JPEG、PNG、WEBP、GIF、PDF、旧 `.xls` 和标准 `.xlsx`；CSV 只拦截明显二进制 NUL。
- 不把本轮表述为病毒扫描、对象存储、图片解码或完整文件安全体系。

## 修改

- `backend/apps/files/services.py` 增加 extension-to-MIME 映射、上传文件头读取和内容签名校验。
- 文件上传测试改用带真实文件头的 JPEG/PNG/WEBP/PDF 样例。
- 新增伪装图片、扩展名/MIME 不一致和 PDF 合法上传测试。
- `.xlsx` 导入坏文件测试改为“ZIP 外壳但非标准 workbook”，保留导入层错误记录覆盖；明显非 ZIP 的 `.xlsx` 由上传阶段拒绝。
- 更新任务图、current-state、README、gap map、backlog、known issues、delivery audit、implementation decisions 和本 Agent run。
