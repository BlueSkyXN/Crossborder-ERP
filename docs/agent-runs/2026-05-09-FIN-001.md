# FIN-001 线下汇款与财务中心

## 输入

- 任务文件：`docs/ai-dev-baseline/agent-execution/tasks/FIN-001.md`
- 关联规格：`docs/source-report-gap-map.md`、`docs/production-readiness-backlog.md`、`docs/ai-dev-baseline/04-api-database-contract.md`
- 人工约束：no-Docker；SQLite 为当前唯一真实验证数据库；PostgreSQL/MySQL/Redis/Celery 后续补齐但本轮不验证；不接真实支付网关。

## Agent 决策

- 复用既有 `RechargeRequest` 作为线下汇款单承载，对外 API 使用 `/remittances` 表达用户线下汇款，避免新增重复财务单据。
- 汇款凭证必须引用 `FILE-001` 产出的会员文件，且 `usage=REMITTANCE_PROOF`、文件有效、归属当前用户。
- 后台审核通过在事务内更新钱包余额并生成 `OFFLINE_REMITTANCE` 钱包流水；已审核汇款单再次审核返回状态冲突，避免重复入账。

## 修改

- 后端扩展 finance 模型、migration、serializer、service、API 和测试，新增用户汇款提交/列表与后台汇款列表/审核通过/取消接口。
- Admin Web 新增财务管理页面，覆盖线下汇款审核、钱包流水和支付单查询。
- User Web 新增 `/finance` 财务中心，支持凭证上传、汇款提交、汇款记录和钱包流水。
- Mobile H5 新增 `/me/finance` 财务中心，并在“我的”页提供入口。
- API 级 E2E 将主链路资金来源改为用户提交线下汇款、后台审核入账后余额支付。
