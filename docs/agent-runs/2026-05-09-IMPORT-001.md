# IMPORT-001 批量导入/导出基础

## 输入

- 任务文件：`docs/ai-dev-baseline/agent-execution/tasks/IMPORT-001.md`
- 关联规格：`docs/source-report-gap-map.md`、`docs/production-readiness-backlog.md`
- 人工约束：no-Docker、SQLite-first；PostgreSQL/MySQL/Redis/Celery 不真实验证；不新增随机依赖。

## Agent 决策

- 仓库后端没有 Excel parser 依赖，本轮使用 Python 标准库 `csv` 实现 CSV fallback。
- `.xls/.xlsx` 只保留为上传类型预留；导入执行阶段返回 CSV-only 的失败任务记录，不把 Excel 原生解析写成已完成。
- 包裹批量预报采用 all-or-none：任何行级错误都会记录失败 job，不创建部分包裹。
- 重复单号按现有 `Parcel.tracking_no` 全局唯一约束校验。
- 模板和导出 CSV 使用即时 HTTP 响应，不持久化生成文件，不提交导出产物。

## 修改

- 后端新增 `ParcelImportJob` 和 migration，记录导入文件、状态、行数、成功数和 `errors_json`。
- 后端新增包裹预报 CSV 模板下载、会员导入、会员导出和后台 RBAC 导出 API。
- 导入校验覆盖仓库代码、快递单号必填/长度、CSV 内重复、系统已存在单号、数量、申报价值、商品字段和 CSV-only 边界。
- User Web 包裹页新增模板下载、CSV 上传导入、导入结果/错误明细、最近导入和会员导出入口。
- Admin Web 包裹入库页新增 `parcels.view` 权限下的 CSV 导出入口。
- `npm run e2e` 的 API 主链路新增 `[IMPORT-001] 批量导入导出链路完成`。
- 更新 README、backend README、gap map、backlog、known issues、deployment、任务状态和下一任务 `QA-BROWSER-001`。
