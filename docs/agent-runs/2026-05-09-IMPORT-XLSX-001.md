# IMPORT-XLSX-001 Excel 批量预报解析

## 背景

完成态审计发现，源报告多处要求“Excel 模板批量导入预报”，而 `IMPORT-001` 只完成 CSV fallback，并明确不声明 Excel 原生解析完成。本轮继续收敛该缺口，同时遵守 no-Docker、SQLite-first、不新增依赖和不污染本地环境的约束。

## 关键实现

- 后端在 `backend/apps/parcels/import_export.py` 中新增标准 `.xlsx` 模板生成和解析能力，基于 Python 标准库 `zipfile` + `xml.etree.ElementTree`，不新增第三方依赖。
- `.xlsx` 导入复用现有 `ParcelImportJob`、表头、行级校验、重复单号校验、错误明细和 all-or-none 事务语义。
- 新增 `/api/v1/parcels/import-template.xlsx` 下载 endpoint，保留原 CSV 模板 endpoint。
- User Web 批量预报入口新增 Excel 模板下载，文件选择支持 `.csv` 和 `.xlsx`。
- Browser Smoke 更新为检查 User Web 批量预报入口暴露 Excel 模板。
- 文档、gap map、production backlog、known issues、delivery audit、implementation decisions 和任务图同步更新。

## 未验证边界

- 旧版二进制 `.xls` 不支持，需另存为 `.xlsx` 或 CSV。
- 不支持复杂公式求值、多 sheet 合并、宏工作簿和第三方 Excel 方言。
- PostgreSQL/MySQL/Redis/Celery/Docker 仍未真实验证。
- 完整业务旅程级浏览器测试、对象存储、真实支付、真实物流 API 和自动采购仍是后续任务。
