# MEMBER-001 后台会员管理增强

## 输入

- 任务文件：`docs/ai-dev-baseline/agent-execution/tasks/MEMBER-001.md`
- 关联规格：`docs/source-report-gap-map.md`、`docs/production-readiness-backlog.md`
- 人工约束：no-Docker、SQLite-first；不接真实短信、邮件、验证码、找回密码、复杂 CRM 或自动分配。

## Agent 决策

- 将会员管理做成后台真实 API 和 Admin Web 页面，不继续使用占位 workspace。
- 在 `MemberProfile` 增加 `assigned_service_admin` 和 `service_note`，用于客服分配占位和内部服务备注。
- 用户端 profile serializer 不暴露后台客服备注和分配字段，避免内部信息泄露。
- 测试密码重置只服务 demo/test 场景，默认仍使用 `password123`。

## 修改

- 后端新增后台会员列表、详情、筛选、资料维护、冻结、解冻、测试密码重置和客服负责人选项 API。
- 后端补会员 RBAC、冻结后已有 member token 阻断、密码重置、客服备注不外泄和 API E2E 测试。
- Admin Web `/members` 升级为真实会员管理页面，支持筛选、状态查看、资料编辑、客服分配、冻结/解冻和测试密码重置。
- `npm run e2e` 增加 `[MEMBER-001] 会员后台链路完成`。
- 更新任务状态、backlog、gap map、README 和部署边界。
