# MSG-001 客服消息与工单入口

## 输入

- 任务文件：`docs/ai-dev-baseline/agent-execution/tasks/MSG-001.md`
- 关联规格：`docs/ai-dev-baseline/13-integrated-product-spec.md`、`14-module-implementation-spec.md`、`04-api-database-contract.md`
- 人工约束：no-Docker、SQLite-first；不接第三方客服、真实在线客服、Redis/Celery 实时推送。

## Agent 决策

- 使用独立 `tickets` app 承载工单和会话消息，状态限定为 `OPEN`、`PROCESSING`、`CLOSED`。
- 附件复用 `FILE-001` 的 `MESSAGE_ATTACHMENT`，用户只能引用自己上传且仍 active 的附件。
- 后台新增独立 `tickets.view` 菜单权限和 `support@example.com` 演示客服账号，不借用会员或财务权限。

## 修改

- 后端新增 tickets 模型、migration、service、serializer、用户 API、后台 API 和权限隔离测试。
- Admin Web 新增 `/tickets` 客服工单页，支持查看、标记处理中、回复和关闭。
- User Web 新增 `/tickets` 消息中心，支持创建工单、上传附件、查看客服回复和补充消息。
- Mobile H5 新增 `/me/tickets`，并在“我的”页加入消息工单入口。
- API E2E 增加用户创建工单、后台回复、用户读取回复的往返验证。
- 更新任务状态、backlog、差距地图、README 和部署边界。
