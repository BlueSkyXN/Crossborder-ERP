# PARCEL-CLAIM-001 无主包裹用户认领

## 输入

- 任务文件：`docs/ai-dev-baseline/agent-execution/tasks/PARCEL-CLAIM-001.md`
- 关联规格：`docs/source-report-gap-map.md`、`docs/production-readiness-backlog.md`
- 人工约束：no-Docker、SQLite-first；PostgreSQL/MySQL/Redis/Celery 不真实验证。

## Agent 决策

- 用户端只返回 `tracking_no_masked`，不暴露完整快递单号和后台内部说明。
- 无主包裹认领先进入 `CLAIM_PENDING`，后台审核通过后才创建会员 `Parcel.IN_STOCK`。
- 认领材料先用 `claim_note` 和 `claim_contact` 支撑人工审核；最终凭证规则、争议处理和通知外呼保留业务确认。
- 后台审核复用 `parcels.view` 权限和现有 WMS 页面，不新增权限体系拆分。

## 修改

- 后端扩展 `UnclaimedParcel` 认领/审核字段，新增用户侧列表/搜索/认领 API 和后台通过/驳回 API。
- 后端服务层补脱敏、事务锁、防重复认领、审核通过转会员包裹和入库记录。
- Admin Web WMS 无主包裹表格增加认领信息、审核备注、通过/驳回 Drawer。
- User Web 新增 `/unclaimed-parcels` 页面和 dashboard 入口。
- Mobile H5 新增 `/ship/unclaimed-parcels` 页面，并从寄件页/我的页进入。
- `npm run e2e` 增加 `[PARCEL-CLAIM-001] 无主包裹认领链路完成`。
- 更新任务状态、backlog、gap map、README、部署和已知边界。
