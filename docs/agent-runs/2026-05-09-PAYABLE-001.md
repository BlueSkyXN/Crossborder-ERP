# 2026-05-09 PAYABLE-001 供应商/成本/应付基础

输入来自 `docs/ai-dev-baseline/agent-execution/tasks/PAYABLE-001.md`，目标是在现有会员钱包、线下汇款和支付单之外，补齐后台供应商、成本类型和应付基础。继续遵守 no-Docker、SQLite-first、本地同步任务边界；本轮不接真实银行、自动打款、外部财务系统或供应商最终结算规则。

本轮在 `backend/apps/finance` 新增 `Supplier`、`CostType`、`Payable` 和 `PayableStatus`，并补迁移 `0003_costtype_supplier_payable.py`。Admin API 覆盖供应商、成本类型和应付款的列表、创建、详情、更新，以及应付款确认、核销和取消。应付款创建后进入 `PENDING_REVIEW`，确认后进入 `CONFIRMED`，核销后进入 `SETTLED`；重复核销显式返回 `STATE_CONFLICT`，避免重复记账。供应商和成本类型停用后不能用于新建应付款。

Admin Web 的财务页已增加 `应付款`、`供应商`、`成本类型` 三个入口，保留既有线下汇款、钱包流水和支付单页签。应付款支持新建、待审核编辑、确认、核销和取消；供应商和成本类型支持新增和编辑。浏览器 smoke 已进入 `/finance` 验证 `财务管理`、`应付款`、`供应商` 和 `新建应付款` 文案。
