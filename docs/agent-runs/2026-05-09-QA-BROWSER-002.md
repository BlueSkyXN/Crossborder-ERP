# QA-BROWSER-002 浏览器真实业务旅程增强

## 背景

完成态审计显示，`QA-BROWSER-001` 已经把三端登录和关键页面 smoke 纳入 CI，但浏览器脚本仍主要停留在页面可见性检查。本轮在不新增依赖、不下载浏览器、不污染用户 Chrome profile 的前提下，补一条真实跨端业务旅程。

## 关键实现

- `scripts/e2e/browser-smoke.mjs` 新增按 label 和 placeholder 填写表单的 CDP DOM helper，并支持点击 Ant Design tab。
- User Web 通过真实包裹预报表单创建唯一 `BRW-*` 快递单号。
- Admin Web 通过真实扫描入库表单扫描同一快递单号、填写重量和尺寸，并验证进入在库列表。
- User Web 再次打开包裹中心，搜索同一快递单号，验证状态为在库且出现申请打包入口。
- 既有 Admin Web、User Web、Mobile H5 登录和关键页面 smoke 保持不变。
