# QA-BROWSER-004 跨面板浏览器业务旅程

## 输入

- 用户目标要求继续补齐原始 ChatGPT/Gemini 报告差距，面板必须真实全栈整合，不能停留在 demo。
- 当前任务图在 `ACCOUNT-RESET-001` 后无自动下一项，但 backlog 和 known issues 仍把更深浏览器业务旅程列为生产差距。
- 约束保持 SQLite-first、no-Docker、不安装全局依赖、不下载浏览器。

## Agent 决策

- 继续复用现有 system Chrome CDP Browser Smoke，不引入 Playwright/Vitest，不下载浏览器。
- 把浏览器验证从包裹预报/入库/回看扩展到财务和客服跨面板链路：User Web 创建线下汇款和客服工单，Admin Web 审核汇款入账并回复工单，User Web 回看客服回复。
- 对浏览器旅程中暴露的真实前后端契约问题直接修复，而不是只调整测试断言。

## 修改

- `scripts/e2e/browser-smoke.mjs`：新增财务/客服跨面板旅程，覆盖浏览器会话内创建汇款、后台审核、创建工单、后台回复、用户回看；新增精确按钮点击 helper，避免误点 Ant Design tab。
- `admin-web/src/features/tickets/api.ts`：修复后台工单回复 URL，从不存在的 `/reply` 改为后端真实契约 `/messages`。
- `admin-web/src/features/finance/FinancePage.tsx`：把汇款审核弹窗的 `Alert` 从 deprecated `message` 改为 `title`，避免 Browser Smoke 捕获 console warning。
- `admin-web/src/features/tickets/TicketOpsPage.tsx`：把 `Timeline` item 从 deprecated `children` 改为 `content`，避免 Browser Smoke 捕获 console warning。
- 更新 README、source report gap map、production backlog、known issues、delivery completion audit、任务图和 current-state。
