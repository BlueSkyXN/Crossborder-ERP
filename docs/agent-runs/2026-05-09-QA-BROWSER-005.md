# QA-BROWSER-005 运单后半程浏览器业务旅程

## 输入

- 用户目标要求继续补齐原始 ChatGPT/Gemini 报告差距，面板必须真实全栈整合，不能停留在 demo。
- 当前 backlog 和 known issues 仍把更深浏览器业务旅程列为生产差距；执行入口也要求 P0 主链路覆盖运单审核、计费、支付、发货、轨迹和确认收货。
- 约束保持 SQLite-first、no-Docker、不安装全局依赖、不下载浏览器。

## Agent 决策

- 继续复用现有 system Chrome CDP Browser Smoke，不引入 Playwright/Vitest，不下载浏览器。
- 在 `QA-BROWSER-004` 的财务入账之后复用同一临时 SQLite 会话和同一浏览器登录态，把已入库包裹继续推进到运单后半程。
- 对 Browser Smoke 捕获到的 Ant Design `useForm` console error 做真实前端修复，而不是在 smoke 中忽略 warning。

## 修改

- `scripts/e2e/browser-smoke.mjs`：新增按 `title` 点击 icon-only 按钮、按文本勾选 checkbox、浏览器会话内查询会员运单等 helper；新增 User Web 创建运单、Admin Web 审核计费、User Web 余额支付、Admin Web 发货、User Web 回看轨迹并确认收货的跨面板旅程。
- `admin-web/src/features/waybills/WaybillOpsPage.tsx`：把运单动作弹窗的表单初始化移到 Modal 打开后的 `useEffect`，避免在 Form 挂载前调用 `setFieldsValue` 触发 Ant Design console error。
- 更新 README、source report gap map、production backlog、known issues、delivery completion audit、implementation decisions、任务图和 current-state。

## 验证

- `node --check scripts/e2e/browser-smoke.mjs`：通过。
- `pnpm --filter admin-web lint`：通过。
- `pnpm --filter admin-web build`：通过。
- `npm run e2e:browser`：passed，使用 `.tmp/browser-e2e` 临时 SQLite、media、日志和 Chrome profile，覆盖包裹预报/入库/回看、财务汇款审核、客服工单回复、运单创建、后台审核计费、用户余额支付、后台发货、用户回看轨迹和确认收货。

## 未验证边界

- 本任务不声明完成 Playwright、组件级测试、视觉回归或所有复杂业务路径覆盖。
- 运单发货仍是本地人工发货和人工轨迹录入，不接真实物流 API、真实面单打印、第三方转单接口或仓库硬件。
- 余额支付仍使用本地钱包余额，不接真实线上支付、退款、渠道对账或支付回调。
- PostgreSQL/MySQL/Redis/Celery/Docker 仍不真实验证。
