# RBAC-DELETE-001 角色与管理员安全删除闭环

## 输入

- 完成度审计显示当前仍不能宣布完整生产级 ERP 完成，`docs/source-report-gap-map.md` 和 `docs/production-readiness-backlog.md` 仍把角色/管理员删除列为后台 RBAC 缺口。
- 用户要求继续补齐原始 ChatGPT/Gemini 报告覆盖范围，同时保持 SQLite-first、no-Docker、不安装全局依赖、不下载浏览器。

## Agent 决策

- 选择本地可验证、无需外部服务的 RBAC 删除闭环，继续收敛后台生产可用性。
- 删除边界采用安全保护：不能删除 `super_admin` 角色，不能删除仍分配给管理员的角色，不能删除内置超级管理员账号，不能删除当前登录管理员账号。
- 删除仍复用现有后台写操作审计链路；不在本任务引入外部 IAM、审批流或更细 create/update/delete 子权限。
- Browser Smoke 原扫描入库步骤连续两次在通用文本点击处触发请求 abort；本轮改为只点击 `button[type=submit]` 的 `clickSubmitButton`，避免误点非提交控件。

## 修改

- `AdminRoleDetailView` 增加 `DELETE /api/v1/admin/roles/{id}`。
- `AdminAccountDetailView` 增加 `DELETE /api/v1/admin/admin-users/{id}`。
- Admin Web `/roles` 表格增加删除按钮和二次确认。
- Admin Web `/admin-users` 表格增加删除按钮和二次确认。
- Browser Smoke 断言 roles/admin-users 删除入口可见，并把扫描入库提交改为 submit button 点击。
- 更新任务图、current-state、README、gap map、backlog、known issues、delivery audit、implementation decisions 和本 Agent run。
