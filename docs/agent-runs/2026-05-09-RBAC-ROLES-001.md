# RBAC-ROLES-001 角色权限管理闭环

## 输入

- 用户要求继续证明纯 AI 驱动开发 ERP，并且每轮任务提交 PR、更新 PR 信息并合并到 `main`。
- `ADMIN-PANELS-001` 已完成后台 dashboard/roles 真实只读面板，但角色编辑和权限分配仍是后续边界。
- 任务文件：`docs/ai-dev-baseline/agent-execution/tasks/RBAC-ROLES-001.md`。

## Agent 决策

- 新增 `iam.role.manage` 作为按钮级写权限，继续保留 `iam.role.view` 负责菜单和只读访问。
- `super_admin` 自动拥有全部菜单和按钮权限，但内置 `super_admin` 角色本身不可被编辑。
- 先补角色创建、编辑和权限集合分配，不做角色删除或后台用户分配角色，避免扩大 IAM 风险面。

## 修改

- 后端新增权限列表 endpoint、角色创建 endpoint 和角色详情/更新 endpoint。
- 角色写入 serializer 校验角色编码唯一性、权限编码存在性和重复权限去重。
- Admin Web `/roles` 增加新增/编辑弹窗、权限勾选列表和只读提示。
- Admin layout 将当前用户的完整权限码传给页面，前端据此控制 `iam.role.manage` 操作入口。
- Browser Smoke 增加 `/roles` 页面 `新增角色` 文本断言。
