# 2026-05-09 SHIP-BATCH-001 发货批次/转单/打印模板数据

输入来自 `docs/ai-dev-baseline/agent-execution/tasks/SHIP-BATCH-001.md`，目标是补齐后台 WMS/运单管理里的发货批次、转单号和打印模板数据基础。继续遵守 no-Docker、SQLite-first、本地同步任务边界；本轮不接真实打印机、物流 API、第三方转单接口或仓库硬件。

本轮在 `backend/apps/waybills` 新增 `ShippingBatch` 和批次状态，给 `Waybill` 增加 `shipping_batch`、`transfer_no` 字段，并补迁移 `0003_waybill_transfer_no_shippingbatch_and_more.py`。后端 Admin API 覆盖 `GET/POST /api/v1/admin/shipping-batches`、详情、归批/移出、锁定、批量发货、批量轨迹和 `print-data`/`print-preview`。批次只允许纳入 `PENDING_SHIPMENT` 且同仓运单，锁定后才能批量发货；重复调用已发货批次发货接口保持幂等，不重复创建关键发货轨迹。

Admin Web 在 `WaybillOpsPage` 内增加发货批次工作区，支持批次列表、创建批次、查看详情、归批、移出、锁定、批量发货、批量轨迹和面单/拣货单/交接单模板数据预览。打印入口只展示结构化数据，包括批次、运单、包裹、收件快照、转单号、承运商批次号和重量汇总，不声明真实硬件打印完成。
