# Agent Runs

本目录用于记录 AI Agent 的关键执行证据，但不记录逐轮对话和细碎过程。

记录粒度：

- 每个正式任务 ID 最多一份，例如 `BE-001`。
- 阶段里程碑可单独一份，例如 `P1_IDENTITY_CONFIG-summary`。
- 纯文档小修、`.gitignore` 小修、清理本地元数据等维护动作，不需要单独记录。
- 不粘贴完整日志、完整 diff、长篇过程复盘。

建议文件名：

```text
YYYY-MM-DD-<task-id>.md
```

每份记录至少包含：

- 输入：任务文件、关联规格、人工约束。
- Agent 决策：只写关键方案取舍。
- 修改：写范围摘要，不逐文件解释。
- 验证：写命令和结果摘要，不粘贴完整日志。
- 未验证边界：尤其是 `configured_unverified` 能力。
- 下一步：后续任务或风险。

建议长度：普通任务控制在 300-800 中文字；复杂任务允许略长，但应保持摘要性质。
