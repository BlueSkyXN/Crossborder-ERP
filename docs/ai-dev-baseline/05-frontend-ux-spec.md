# 05 前端与交互设计规格

## 总原则

三端都要重新设计，不复刻旧系统视觉。可以参考业务信息结构，但不能复制布局、图标、图片、文案和品牌风格。

第一版 UI 目标：

- 后台端重效率：表格、筛选、批量操作、状态标签、详情抽屉。
- 用户 Web 重闭环：仓库地址、包裹、运单、支付、代购入口清晰。
- 移动 H5 重任务：复制地址、预报包裹、查包裹、申请打包、查运单。

## 设计系统

统一设计 token：

| 类型 | 规则 |
| --- | --- |
| 主色 | 选一个独立品牌色，不使用旧系统配色 |
| 字体 | 系统默认字体，中文优先可读性 |
| 圆角 | 管理端 4-8px，移动端 8-12px |
| 状态色 | 成功、警告、错误、处理中、已取消统一映射 |
| 间距 | 4/8/12/16/24/32 体系 |
| 图标 | 使用授权图标库，例如 lucide-react 或 Ant Design Icons |
| 空状态 | 每个列表必须有空状态 |
| 错误态 | 每个接口失败必须有错误提示和重试 |
| 加载态 | 表格、按钮、详情页必须有 loading |

### 后台通用页面模板

列表页必须包含：

- 页面标题。
- 操作按钮区，例如新增、导出、批量删除。
- 筛选区：关键词、状态、日期、仓库。
- 表格。
- 分页。
- 空状态。
- 错误态。

详情页建议采用：

- 左侧或顶部状态条。
- 基本信息区。
- 明细表格区。
- 费用区。
- 操作日志区。
- 右侧操作按钮或底部固定操作栏。

## 前端验收要求

每个页面完成时必须满足：

- 有 loading、empty、error 状态。
- 有接口失败提示。
- 表单有必填校验。
- 金额、重量、体积格式正确。
- 列表筛选后分页正确。
- 状态按钮只在允许状态显示。
- 移动端底部安全区不遮挡按钮。
- 不出现旧系统文案和视觉复刻。

## UI 原型策略

新手和 Agent 不要一开始追求高保真。建议顺序：

1. 低保真流程图和页面字段。
2. 通用组件模板。
3. 后台先做可用表格和详情。
4. 用户 Web 先跑通业务闭环。
5. 移动 H5 再做响应式和触控体验。

每个页面先写清楚：

```text
页面目标
进入条件
核心字段
主要操作
状态按钮
空状态
错误状态
验收标准
```
