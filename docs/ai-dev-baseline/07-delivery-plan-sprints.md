# 07 端到端交付计划与 Sprint 拆分

## 总体里程碑

```text
Sprint 0  工程启动和基础设施
Sprint 1  账号、权限、仓库、基础配置
Sprint 2  包裹预报、扫描入库、包裹管理
Sprint 3  运单、计费、钱包支付、物流轨迹
Sprint 4  最小代购和商品能力
Sprint 5  三端联调、测试、部署、验收
```

不要跳过 Sprint 0。没有工程规范和基础设施，后面 Agent 会越写越乱。

## Sprint 0：工程启动

目标：创建可运行、可测试、可部署的项目骨架。

后端任务：

- 初始化 Django/DRF。
- 配置 PostgreSQL、Redis。
- 建立 settings 分层。
- 建立 `common` app。
- 统一响应、异常、分页。
- 配置 drf-spectacular。
- 配置 pytest。
- 创建健康检查接口。

前端任务：

- 初始化 `admin-web`、`user-web`、`mobile-h5`。
- 配置 TypeScript、lint、format。
- 配置 API client。
- 配置路由和基础 Layout。
- 配置环境变量。

基础设施：

- `docker-compose.yml`。
- `.env.example`。
- README 本地启动步骤。
- 种子数据命令占位。

验收：

- 后端 `GET /api/v1/health` 返回成功。
- 三个前端能启动并显示基础页面。
- OpenAPI 文档能打开。
- 测试命令能运行。

## Sprint 1：账号、权限、仓库、基础配置

目标：系统可以登录，后台可以配置基础数据，用户能获取仓库地址。

后端：

- `iam`：后台管理员、角色、权限、菜单。
- `members`：用户、会员资料、用户登录。
- `warehouses`：仓库、仓库地址、国家地区。
- 基础配置：包装方式、增值服务、发货渠道。
- 种子数据：管理员、测试用户、仓库、渠道。

后台端：

- 管理员登录。
- 后台 Layout 和菜单。
- 会员列表和详情。
- 仓库配置。
- 发货渠道、包装、增值服务配置。

用户 Web：

- 用户登录。
- 控制台。
- 仓库地址展示和复制。

移动 H5：

- 用户登录。
- 我的页基础。
- 寄件首页仓库地址。

验收：

- 不同后台角色看到不同菜单。
- 用户登录后看到自己的会员收件标识。
- 仓库地址配置后用户端可复制。

## Sprint 2：包裹预报、扫描入库、包裹管理

目标：包裹从预报到入库到在库可见。

后端：

- `parcels` models、services、API。
- 包裹预报。
- 待入库查询。
- 扫描入库。
- 包裹图片。
- 无主包裹简版。

后台端：

- 待入库列表。
- 扫描入库页面。
- 包裹详情。
- 无主包裹管理。

用户 Web：

- 发布包裹预报。
- 包裹列表。
- 包裹详情。

移动 H5：

- 发布预报。
- 包裹详情。
- 我的页包裹入口。

验收：

- 用户提交预报后后台可查。
- 后台入库后用户端状态变为在库。
- 重复入库被拦截。
- 无主包裹可以后台登记并人工归属。

## Sprint 3：运单、计费、钱包支付、物流轨迹

目标：跑通集运核心闭环。

后端：

- `waybills` models、services、API。
- 申请打包发货。
- 运单审核。
- 费用计算和后台调整。
- `finance` 钱包、流水、支付单。
- 余额支付。
- 物流轨迹。

后台端：

- 运单列表。
- 运单详情。
- 审核、计费、发货、问题单。
- 钱包人工充值/扣减。
- 轨迹节点录入。

用户 Web：

- 申请打包。
- 运单列表/详情。
- 余额支付。
- 物流轨迹。

移动 H5：

- 申请打包。
- 运单列表。
- 运单追踪。
- 确认收货。

验收：

- 在库包裹可以生成运单。
- 运单费用生成后用户余额支付。
- 支付后钱包流水正确。
- 后台发货后用户可看到轨迹。
- 用户确认收货后状态结束。

## Sprint 4：最小代购和商品能力

目标：代购订单能生成，并能转入包裹链路。

后端：

- `products` 分类、商品、SKU。
- `purchases` 代购订单、订单明细、采购任务。
- 手工代购。
- 自营商品最小购物车和确认订单。
- 到货转包裹。

后台端：

- 商品分类、商品、SKU。
- 代购订单列表和详情。
- 审核、采购、到货、异常、取消。

用户 Web：

- 商品列表/详情。
- 购物车。
- 手工代购表单。
- 代购订单列表/详情。

移动 H5：

- 首页商品入口。
- 商品详情和购物车简版。
- 代购订单状态入口。

验收：

- 用户可提交手工代购。
- 后台可审核和采购。
- 到货后能生成 Parcel。
- Parcel 继续走集运流程。

## Sprint 5：联调、测试、部署、验收

目标：形成可以交付给别人演示和继续开发的完整项目。

联调：

- 后台、用户 Web、移动 H5 共用同一后端。
- 所有状态枚举一致。
- 所有金额和流水一致。
- 所有关键流程有测试数据。

测试：

- 后端单元测试。
- API 集成测试。
- 前端基础组件测试。
- Playwright 端到端测试。
- 权限和越权测试。

部署：

- Docker Compose 启动。
- staging 环境部署说明。
- 数据库迁移说明。
- 静态资源和媒体文件说明。

交付文档：

- README。
- API 文档链接。
- 测试账号。
- 演示脚本。
- 已知问题。
- 下一阶段计划。

验收：

- 新机器按 README 可启动。
- 演示脚本可完整跑通。
- 所有 P0 验收点通过。
- 未完成事项都列入 backlog。

## 版本发布标准

v0.1-alpha：

- 工程可启动。
- 登录、仓库、预报、入库基础可用。

v0.1-beta：

- 集运主链路跑通。
- 钱包支付可用。

v0.1：

- 最小代购接入。
- 三端联调完成。
- 有测试和部署文档。

## 禁止的排期方式

不要这样排：

```text
第一周做完整后台
第二周做完整用户端
第三周做完整移动端
```

应该按业务闭环排：

```text
先做账号和配置
再做包裹
再做运单和支付
再做代购
最后补体验和部署
```
