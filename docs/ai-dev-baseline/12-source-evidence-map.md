## 总体产品判断证据

ChatGPT 后台分析判断后台属于“跨境代购 + 集运转运 + 仓储入库 + 运单发货 + 会员财务管理”的综合型 B 端运营系统，并列出代购订单、包裹仓储、集运运单、会员、商品、财务、基础配置、内容管理等业务域（`ChatGPT分析/admin分析.md:243-260`）。

后台最核心的模块按重要性包括代购订单管理、集运运单管理、包裹与仓储管理、会员管理、商品管理、财务应收/应付、基础配置、内容管理、应用扩展（`ChatGPT分析/admin分析.md:262-278`）。

用户 Web 分析把前台划分为用户工作台/会员中心、商城/商品代购前台、代购订单、集运/转运、财务/支付、推广/返利、账户设置、服务中心/客服八个业务区（`ChatGPT分析/userweb分析.md:125-139`）。

移动端分析确认底部五栏为首页、分类、寄件、购物车、我的，并将核心任务归纳为商品代购、仓库地址复制、包裹预报、包裹打包、运单追踪、账户财务设置（`ChatGPT分析/usermobile分析.md:155-181`）。

外部调研把公开能力拆为会员账户、代购商品接入、代购订单、自营商城、包裹预报、WMS 仓储、集运订单、直邮/极速包裹、拼邮、运价计费、财务、营销运营 12 个业务域（`跨境代购与集运物流 ERP 系统 调研/同类开源项目调研.md:153-176`）。

## 三端核心流程证据

### 后台端

后台核心流程总表覆盖管理员权限、会员管理、商品管理、代购订单、包裹预报、仓库入库、无主包裹、合箱打包、运单发货、支付财务、费用配置、内容管理（`ChatGPT分析/admin分析.md:480-500`）。

后台关键状态流转包括代购订单待付款/待审核/待采购/已采购/已完成/异常单/订单关闭，运单待打包/待付款/待发货/已发货/已取消/问题单，包裹待入库/在库/无主待认领/已认领，汇款、应付款、内容显示、商品推荐、发货渠道启用等（`ChatGPT分析/admin分析.md:501-515`）。

后台 MVP 必做包括后台基础框架、通用列表能力、基础配置、会员管理、商品管理、代购订单核心流、包裹仓储、运单主流程、批次发货、财务核心、内容基础（`ChatGPT分析/admin分析.md:922-937`）。

Gemini 后台分析进一步把 MVP 收敛为资金账户闭环、基础配置、代购业务、转运核心流，并把 CMS、自动化对接、财务成本模块作为第二阶段或后续（`Gemini分析/admin产品分析.md:504-527`）。

### 用户 Web

用户 Web 买商品/代购路径为商品搜索、商品列表、商品详情、规格选择、加购/购买、代购订单、支付；分析明确购物车页、确认订单页、商城订单支付页在截图中未完整展示，需要独立设计（`ChatGPT分析/userweb分析.md:140-168`）。

用户 Web 集运发货路径为获取仓库地址、发布集运预报、提交直邮运单、批量导入预报、包裹管理/详情、运单管理、运单详情、运费支付、运单追踪（`ChatGPT分析/userweb分析.md:155-168`）。

用户 Web MVP 必做包括用户账户、控制台、仓库地址、商品搜索/列表/详情、购物车、订单、代购订单、万能代购、包裹预报、包裹管理、运单管理、运费支付、财务、收件地址、消息（`ChatGPT分析/userweb分析.md:769-790`）。

用户 Web 阶段结论明确两个闭环：买商品/代购闭环，以及集运发货闭环，并指出登录/注册、购物车、确认订单、支付成功、物流轨迹、地址新增/编辑、运单修改等页面需补齐（`ChatGPT分析/userweb分析.md:963-973`）。

Gemini 用户 Web 分析强调 MVP 要覆盖用户模块、代购模块、集运模块、财务模块、商城基础，同时将复杂 SKU、线下汇款、批量预报、无主包裹、客服作为第二阶段（`Gemini分析/userweb产品分析.md:446-477`）。

### 移动 H5

移动端商城购物链路包括首页搜索/推荐、分类、商品卡片、商品详情、规格、加购/立即购买、购物车、确认订单、代购订单状态入口、登录拦截（`ChatGPT分析/usermobile分析.md:182-198`）。

移动端集运/包裹/运单链路包括集运/直邮模式切换、仓库地址复制、集运指引、发布预报、包裹详情、申请打包、无主包裹、运单列表、运单追踪、我的转运入口（`ChatGPT分析/usermobile分析.md:199-217`）。

移动端流程总表覆盖登录、首页、分类、商品、购物车、确认订单、集运地址、包裹预报、包裹详情、申请打包、无主认领、运单、追踪、我的、财务、设置、客服等 24 条流程（`ChatGPT分析/usermobile分析.md:419-449`）。

移动端 MVP 必做包括底部五栏导航、登录、首页、分类、商品详情、购物车、确认订单、集运首页、集运指引、包裹预报、包裹详情、申请打包、运单列表、运单追踪、我的、设置中心（`ChatGPT分析/usermobile分析.md:888-908`）。

移动端阶段结论同样确认两条主链路：商城/代购链路和集运/包裹/运单链路（`ChatGPT分析/usermobile分析.md:1058-1068`）。

Gemini 移动端分析把 MVP 拆成账号体系、代购主链路、集运主链路、个人中心，并指出外链解析、双币结算、运单计费规则需要业务确认（`Gemini分析/usermobile产品分析.md:482-533`）。

## 净室边界和开源参考证据

开源调研结论是没有一个 GitHub 项目能 1:1 覆盖“代购 + 集运 + 仓储 + 财务 + 多端 + PDA/硬件”，更现实方式是按模块参考代购、WMS、ERP、TMS 项目并独立重构（`跨境代购与集运物流 ERP 系统 调研/同类开源项目调研.md:1-7`）。

调研建议可以模仿业务流程、功能组合、通用业务规则和公开产品模块，但不要模仿 UI 视觉、图标、布局、文案、专有接口、数据库字段、私有代码、部署结构（`跨境代购与集运物流 ERP 系统 调研/同类开源项目调研.md:348-364`）。

外界分析给出的标准集运流程为客户预报、仓管入库、验货打标上架、提交发货指令、拣货、打包计费、付款、出库、海外派送（`跨境代购与集运物流 ERP 系统 调研/外界对金蚁ERP的分析.md:74-97`）。

外界分析给出的代购流程包括抓取代购、自营商城、万能/门店代购，并建议抽象为商品发现、创建代购单、审核报价、付款、采购、异常、分批入库、转在库包裹、集运发货（`跨境代购与集运物流 ERP 系统 调研/外界对金蚁ERP的分析.md:131-155`）。

外界分析建议采用“模块化单体 + 插件化集成”起步，强调代购集运 ERP 的核心复杂度在业务规则与状态一致性，过早微服务化会增加事务复杂度（`跨境代购与集运物流 ERP 系统 调研/外界对金蚁ERP的分析.md:331-372`）。

## 前端复现包证据

Gemini 复现包使用 React 19、Vite、Tailwind CSS、lucide-react、recharts，脚本包含 `dev`、`build`、`lint`（`可学习的ERP风格-Gemini复现/erp/package.json`）。

复现包后台壳包含左侧一级/二级菜单，菜单域包括控制台、代购管理、集运转运、会员管理、商品管理、应收/应付、图片管理、网站管理、基础设置（`可学习的ERP风格-Gemini复现/erp/src/App.tsx:21-51`）。

复现包实现了后台暗色侧栏、顶部导航、可关闭标签页、刷新按钮和内容区切换（`可学习的ERP风格-Gemini复现/erp/src/App.tsx:99-247`）。

复现包表格页体现了管理端高频模式：操作按钮、Excel 导入区、筛选条件、搜索按钮、固定表头、批量勾选、行级编辑、排序、分页（`可学习的ERP风格-Gemini复现/erp/src/pages/TableView.tsx:7-117`）。

复现包控制台体现了用户工作台模式：账户卡、余额/积分卡、订单状态圆形入口、仓库地址字段复制、一键复制、交易图表（`可学习的ERP风格-Gemini复现/erp/src/pages/DashboardView.tsx:22-198`）。

复现包移动模拟器体现了移动端模式：375x812 设备框、顶部返回、集运/直邮 Tab、仓库 Tab、地址详情、一键复制、流程提示、发布预报/申请打包两个主按钮、底部五栏导航（`可学习的ERP风格-Gemini复现/erp/src/pages/MobileSimulator.tsx:6-108`）。

注意：复现包内仍含 KINGANT 标识和旧系统文案，后续实现只能抽象布局和交互模式，不能复用品牌、文案、图片、配色和视觉细节。
