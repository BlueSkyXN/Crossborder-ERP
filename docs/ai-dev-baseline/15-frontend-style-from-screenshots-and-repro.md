# 15 前端风格、布局与交互设计指南

本文件基于截图和 Gemini 前端复现包提炼可学习的布局与交互模式。注意：只能学习信息架构、组件模式和业务交互，不复用旧品牌、Logo、文案、图片、颜色组合和页面视觉。

## 证据来源

- 后台截图：66 页 PDF，66 张 PNG，桌面宽屏管理系统。
- 用户 Web 截图：34 页 PDF，34 张 PNG，用户控制台 + 商品/代购 + 集运中心。
- 移动端截图：21 页 PDF，移动 H5 竖屏。
- 复现包：`可学习的ERP风格-Gemini复现/erp`。
- 复现包技术：React 19、Vite、Tailwind CSS、lucide-react、recharts。

## 总体设计取舍

后续项目采用全新视觉：

- 管理端：Ant Design 风格，低装饰、高密度、高效率。
- 用户 Web：现代会员中心 + 集运工作台 + 商品购物流程。
- 移动 H5：底部五栏、卡片列表、固定底部操作栏。

不要照抄：

- KINGANT/金蚁 Logo。
- 旧系统黑橙配色。
- 旧系统文案和促销图。
- 旧系统页面布局的精确比例。
- 旧系统图标。

可以学习：

- 后台“左侧菜单 + 顶部导航 + 多标签工作区 + 表格”的 ERP 操作模式。
- 用户中心“账户卡 + 状态入口 + 仓库地址复制 + 交易图表”的信息组织。
- 移动端“集运/直邮 Tab + 仓库 Tab + 地址复制 + 两个主按钮 + 底部导航”的任务结构。

## 后台管理端布局

推荐结构：

```text
AppShell
  ├── Sidebar：一级模块 + 二级菜单
  ├── Header：搜索/快捷入口/通知/账户
  ├── WorkspaceTabs：可选，适合后台多任务
  └── Content：列表/详情/表单/看板
```

复现包证据：

- 菜单域包括控制台、代购管理、集运转运、会员管理、商品管理、应收/应付、图片管理、网站管理、基础设置（`src/App.tsx:21-51`）。
- 后台壳体有侧栏、顶部导航、标签页、刷新按钮和内容区切换（`src/App.tsx:99-247`）。
- 表格页包含操作区、导入筛选区、表格、分页（`src/pages/TableView.tsx:7-117`）。

后台设计规则：

- 列表页要支持筛选、分页、状态 Tab、批量操作、导出。
- 详情页要分区显示基础信息、业务明细、费用、状态历史、操作日志。
- 状态变更按钮放在详情页顶部或右侧固定区。
- 高风险操作二次确认。
- 金额、状态、运单号、包裹号要可复制。
- 所有列表要有空状态、加载态、错误态。

后台页面模板：

```text
PageHeader
FilterBar
ActionBar
StatusTabs
DataTable
Pagination
DetailDrawer / DetailPage
AuditLogPanel
```

## 用户 Web 布局

推荐结构：

```text
Public/Mall Layout
  ├── Header：搜索、购物车、账户
  ├── Product/List/Detail
  └── Footer/Help

Member Center Layout
  ├── SideMenu
  ├── TopBar
  └── Content
```

复现包证据：

- 控制台展示账户卡、余额/积分、订单状态、仓库地址、一键复制、交易图表（`src/pages/DashboardView.tsx:22-198`）。

用户 Web 设计规则：

- 控制台首屏必须有仓库地址复制，这是集运主链路入口。
- 状态入口要聚合代购订单、包裹、运单。
- 商品详情要固定购买动作区。
- 包裹/运单列表以表格为主，详情可用页面或抽屉。
- 支付弹窗要突出金额、单号、余额、确认按钮。
- 财务流水要可按类型、时间、业务单号筛选。

## 移动 H5 布局

推荐结构：

```text
MobileApp
  ├── TopBar：标题/返回/搜索
  ├── Content：卡片、Tab、表单
  ├── BottomActionBar：关键提交
  └── BottomTabs：首页/分类/寄件/购物车/我的
```

复现包证据：

- 移动模拟器采用 375x812 设备框和顶部状态栏（`src/pages/MobileSimulator.tsx:6-17`）。
- 有集运/直邮顶部 Tab 和仓库横向 Tab（`src/pages/MobileSimulator.tsx:19-42`）。
- 地址详情、一键复制、流程提示、发布包裹预报/申请打包两个主按钮（`src/pages/MobileSimulator.tsx:44-76`）。
- 底部五栏导航：首页、分类、寄件、购物车、我的（`src/pages/MobileSimulator.tsx:80-107`）。

移动端设计规则：

- 一级入口固定底部五栏。
- 寄件页是集运主入口，不要埋太深。
- 长列表用卡片，不用桌面表格。
- 关键提交流程使用固定底部操作栏。
- 状态 Tab 横向滚动。
- 搜索框放在列表顶部。
- 手机端按钮高度不低于 44px。
- 底部操作栏要避开安全区。

## 实现建议

后台：

- 使用 Ant Design 的 `Layout`、`Menu`、`Table`、`Form`、`Drawer`、`Modal`、`Tabs`。
- 状态标签封装 `StatusTag`。
- 筛选栏封装 `FilterBar`。
- 所有表格封装统一分页和错误处理。

用户 Web：

- 用自定义轻量组件，避免过重后台感。
- 控制台和业务中心可复用卡片、表格、状态入口。

移动 H5：

- 可选 Ant Design Mobile 或自研基础组件。
- 必须真实移动视口验收。

## 视觉禁区

后续 Agent 不得：

- 使用 KINGANT 或金蚁字样。
- 复用旧截图里的商品广告图。
- 使用旧系统 logo。
- 复制旧文案。
- 原样复刻旧系统黑橙视觉。
- 在用户 Web 或移动端做成后台表格堆砌。

可以：

- 使用同类业务结构。
- 重写文案。
- 自定义品牌色。
- 使用授权图标和图片。
- 用现代组件库重新设计。
