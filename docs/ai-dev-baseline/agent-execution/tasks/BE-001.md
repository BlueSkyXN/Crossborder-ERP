# BE-001 初始化后端项目

phase: `P0_FOUNDATION`  
depends_on: `INIT-001`  
next: `BE-002`

## 目标

初始化 Django + DRF 后端，建立统一响应、错误、分页、OpenAPI 和测试基线。

## 必读

- `../../03-technical-architecture.md`
- `../../04-api-database-contract.md`
- `../../14-module-implementation-spec.md`

## 必须做

- 创建 Django 项目和 `config/settings/` 分层。
- 安装并配置 `Django REST Framework`、`drf-spectacular`、`pytest`。
- 创建 `apps/common/`。
- 实现 `GET /api/v1/health`。
- 实现统一成功响应、错误响应、异常处理、分页基类。
- 让 OpenAPI 文档可访问。

## 不要做

- 不实现业务 app。
- 不提前设计复杂微服务。

## 验收

```bash
cd backend
python manage.py check
pytest
```

接口验收：

- `GET /api/v1/health` 返回 `{"code":"OK","message":"success","data":...}`。
- OpenAPI schema 能生成。
