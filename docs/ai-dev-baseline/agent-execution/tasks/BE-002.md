# BE-002 后台 IAM 和 RBAC

phase: `P1_IDENTITY_CONFIG`  
depends_on: `BE-001`  
next: `BE-003`

## 目标

实现后台管理员登录、角色权限和菜单权限骨架。

## 必读

- `../../02-domain-model-state-machines.md`
- `../../04-api-database-contract.md`
- `../../14-module-implementation-spec.md`

## 必须做

- Django app：`iam`
- Models：`AdminUser`、`Role`、`Permission`
- APIs：
  - `POST /api/v1/admin/auth/login`
  - `GET /api/v1/admin/me`
  - `GET /api/v1/admin/menus`
  - `GET /api/v1/admin/roles`
- 创建超级管理员、仓库人员、财务人员、采购人员种子数据。
- 后台接口和用户端接口权限域分离。

## 不要做

- 不把普通会员用户当管理员。
- 不只隐藏前端按钮，必须有后端权限校验。

## 验收

```bash
cd backend
pytest apps/iam
```

接口验收：

- 超级管理员可登录。
- 无 token 返回 `UNAUTHORIZED`。
- 无权限返回 `FORBIDDEN`。
