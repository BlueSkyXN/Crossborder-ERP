# BE-003 用户和会员资料

phase: `P1_IDENTITY_CONFIG`  
depends_on: `BE-002`  
next: `BE-004`

## 目标

实现会员注册、登录、当前用户、会员资料和收件标识。

## 必读

- `../../13-integrated-product-spec.md`
- `../../14-module-implementation-spec.md`
- `../../04-api-database-contract.md`

## 必须做

- Django app：`members`
- Models：`User`、`MemberProfile`
- APIs：
  - `POST /api/v1/auth/register`
  - `POST /api/v1/auth/login`
  - `POST /api/v1/auth/logout`
  - `GET /api/v1/me`
  - `PUT /api/v1/me/profile`
- 自动生成会员收件标识。
- 创建测试用户种子数据。
- 用户数据隔离测试。

## 不要做

- 不实现复杂验证码和找回密码，标记 `TODO_CONFIRM`。
- 不让用户 token 访问后台接口。

## 验收

```bash
cd backend
pytest apps/members
```

接口验收：

- 测试用户可登录。
- `GET /api/v1/me` 返回会员资料和收件标识。
- 用户 A 不能访问用户 B 的数据。
