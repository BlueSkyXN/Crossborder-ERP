# BE-004 仓库和基础配置

phase: `P1_IDENTITY_CONFIG`  
depends_on: `BE-003`  
next: `FEA-001`

## 目标

实现集运业务运行所需的仓库、仓库地址、渠道、包装、增值服务和简版费率。

## 必读

- `../../13-integrated-product-spec.md`
- `../../14-module-implementation-spec.md`
- `../../04-api-database-contract.md`

## 必须做

- Django apps：`warehouses`，必要时在 `waybills` 中放渠道/费用模型。
- Models：`Warehouse`、`WarehouseAddress`、`ShippingChannel`、`RatePlan`、`PackagingMethod`、`ValueAddedService`
- 用户端 APIs：
  - `GET /api/v1/warehouses`
  - `GET /api/v1/warehouses/{id}/address`
- 后台 CRUD APIs：
  - `/api/v1/admin/warehouses`
  - `/api/v1/admin/shipping-channels`
  - `/api/v1/admin/packaging-methods`
  - `/api/v1/admin/value-added-services`
  - `/api/v1/admin/rate-plans`
- 种子数据：深圳仓、北京仓、测试空运、测试海运、纸箱、编织袋、拍照、加固、防水。

## 不要做

- 不实现复杂体积重/偏远费/保险费公式，先保留 `TODO_CONFIRM`。
- 不做多仓智能分配。

## 验收

```bash
cd backend
pytest apps/warehouses
```

业务验收：

- 后台创建仓库后，用户端能看到带会员收件标识的专属仓库地址。
- 停用渠道不能被新运单选择。
