# BE-005 包裹预报和扫描入库

phase: `P2_PARCEL_WMS`  
depends_on: `BE-003`, `BE-004`  
next: `FEA-003`

## 目标

实现 Parcel 从用户预报到后台扫描入库，再到在库可发货。

## 必读

- `../../02-domain-model-state-machines.md`
- `../../04-api-database-contract.md`
- `../../14-module-implementation-spec.md`

## 必须做

- Django app：`parcels`
- Models：`Parcel`、`ParcelItem`、`ParcelPhoto`、`InboundRecord`、`UnclaimedParcel`
- Services：`forecast_parcel`、`scan_inbound`、`claim_unclaimed_parcel`
- 用户端 APIs：
  - `POST /api/v1/parcels/forecast`
  - `GET /api/v1/parcels`
  - `GET /api/v1/parcels/{id}`
  - `GET /api/v1/parcels/packable`
- 后台 APIs：
  - `GET /api/v1/admin/parcels`
  - `POST /api/v1/admin/parcels/scan-inbound`
  - `POST /api/v1/admin/parcels/{id}/inbound`
  - `GET/POST /api/v1/admin/unclaimed-parcels`
- 状态机：`PENDING_INBOUND -> IN_STOCK`。

## 不要做

- 不用 `Package` 作为核心对象名。
- 不做复杂库位/PDA/电子秤集成。

## 验收

```bash
cd backend
pytest apps/parcels
```

业务验收：

- 预报后状态 `PENDING_INBOUND`。
- 入库后状态 `IN_STOCK`。
- 重复入库返回 `STATE_CONFLICT`。
- 用户不能访问别人包裹。
