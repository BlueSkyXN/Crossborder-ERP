# BE-006 运单和打包

phase: `P3_WAYBILL_FINANCE`  
depends_on: `BE-005`  
next: `BE-007`

## 目标

实现 Waybill 从申请打包、后台审核、设置费用到待付款。

## 必读

- `../../02-domain-model-state-machines.md`
- `../../04-api-database-contract.md`
- `../../14-module-implementation-spec.md`

## 必须做

- Django app：`waybills`
- Models：`Waybill`、`WaybillParcel`
- Services：`create_waybill`、`review_waybill`、`set_waybill_fee`、`cancel_waybill`
- 用户 APIs：
  - `POST /api/v1/waybills`
  - `GET /api/v1/waybills`
  - `GET /api/v1/waybills/{id}`
- 后台 APIs：
  - `GET /api/v1/admin/waybills`
  - `POST /api/v1/admin/waybills/{id}/review`
  - `POST /api/v1/admin/waybills/{id}/set-fee`
- 只有 `IN_STOCK` Parcel 可创建 Waybill。
- 创建后 Parcel 变 `PACKING_REQUESTED`。

## 不要做

- 不用 `Shipment` 作为核心对象名。
- 不在 view 里直接改状态。

## 验收

```bash
cd backend
pytest apps/waybills
```

业务验收：

- 创建运单后状态为 `PENDING_REVIEW`。
- 后台审核设置费用后状态进入 `PENDING_PAYMENT`。
- 非法状态流转返回 `STATE_CONFLICT`。
