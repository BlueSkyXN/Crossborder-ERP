# BE-007 钱包和余额支付

phase: `P3_WAYBILL_FINANCE`  
depends_on: `BE-006`  
next: `BE-008`

## 目标

实现 Wallet、PaymentOrder、WalletTransaction，支持后台充值和用户余额支付。

## 必读

- `../../04-api-database-contract.md`
- `../../14-module-implementation-spec.md`
- `../../08-qa-security-deployment.md`

## 必须做

- Django app：`finance`
- Models：`Wallet`、`WalletTransaction`、`PaymentOrder`、`RechargeRequest`
- Services：`admin_recharge`、`admin_deduct`、`create_payment_order`、`pay_with_wallet`
- APIs：
  - `GET /api/v1/wallet`
  - `GET /api/v1/wallet/transactions`
  - `POST /api/v1/waybills/{id}/pay`
  - `GET /api/v1/admin/wallet-transactions`
  - `POST /api/v1/admin/users/{id}/wallet/recharge`
  - `POST /api/v1/admin/users/{id}/wallet/deduct`
  - `GET /api/v1/admin/payment-orders`
- 钱包扣款和业务状态推进必须在同一事务。
- 支付必须幂等。

## 不要做

- 不接真实在线支付回调。
- 不用 float 表示金额。

## 验收

```bash
cd backend
pytest apps/finance
```

业务验收：

- 后台充值生成流水。
- 用户余额支付扣款并推进 Waybill。
- 重复支付不重复扣款。
- 余额不足返回 `INSUFFICIENT_BALANCE`。
