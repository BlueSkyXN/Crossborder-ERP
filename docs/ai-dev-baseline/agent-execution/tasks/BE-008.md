# BE-008 物流轨迹

phase: `P3_WAYBILL_FINANCE`  
depends_on: `BE-006`  
next: `FEA-004`

## 目标

实现 TrackingEvent，支持后台人工添加轨迹和用户查询轨迹。

## 必读

- `../../02-domain-model-state-machines.md`
- `../../04-api-database-contract.md`
- `../../14-module-implementation-spec.md`

## 必须做

- Model：`TrackingEvent`
- Services：`ship_waybill`、`add_tracking_event`、`confirm_receipt`
- APIs：
  - `GET /api/v1/waybills/{id}/tracking-events`
  - `GET /api/v1/waybills/tracking?waybill_no=xxx`
  - `POST /api/v1/waybills/{id}/confirm-receipt`
  - `POST /api/v1/admin/waybills/{id}/ship`
  - `POST /api/v1/admin/waybills/{id}/tracking-events`
- 发货时至少创建一个轨迹节点。
- 轨迹按时间排序。

## 不要做

- 不接真实物流 API 自动同步。
- 不暴露其他用户私有运单轨迹。

## 验收

```bash
cd backend
pytest apps/waybills
```

业务验收：

- 后台发货后 Waybill 进入 `SHIPPED`。
- 用户查看轨迹并确认收货后 Waybill 进入 `SIGNED`。
