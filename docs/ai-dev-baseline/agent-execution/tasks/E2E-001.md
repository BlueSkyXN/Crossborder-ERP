## 目标

用 Playwright 或可复现手工脚本验证 P0 端到端闭环。

## 必读

- `../../08-qa-security-deployment.md`
- `../../16-end-to-end-agent-runbook.md`

## 必须做

- 准备 demo seed 数据和测试账号。
- 覆盖后台、用户 Web、移动 H5 至少一个主链路。
- 优先写 Playwright E2E。
- 如暂时无法全自动，必须写可复现手工脚本。

## 主链路

```text
后台配置仓库/渠道/包装/增值服务
-> 用户登录并复制仓库地址
-> 用户提交包裹预报
-> 后台扫描入库
-> 用户申请打包
-> 后台审核运单并设置费用
-> 后台给用户充值
-> 用户余额支付
-> 后台发货并添加轨迹
-> 用户查看轨迹并确认收货
```

## 代购链路

```text
用户提交手工代购
-> 后台审核、采购、到货
-> 到货转 Parcel
-> Parcel 可继续申请打包
```

## 验收

```bash
npm run e2e
```

如果命令不同，必须在根 README 固化。

验收输出必须包含：

- 测试账号。
- 启动命令。
- E2E 命令。
- 失败时的阻塞项。
