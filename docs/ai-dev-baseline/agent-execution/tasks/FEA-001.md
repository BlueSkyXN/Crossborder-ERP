# FEA-001 后台登录和 Layout

phase: `P1_IDENTITY_CONFIG`  
depends_on: `BE-002`  
next: `FEA-002`

## 目标

实现后台管理端基础壳：登录、API client、路由守卫、Layout、菜单。

## 必读

- `../../15-frontend-style-from-screenshots-and-repro.md`
- `../../04-api-database-contract.md`
- `../../03-technical-architecture.md`

## 必须做

- 初始化或完善 `admin-web`。
- 配置 API client，统一处理 `code/message/data`。
- 实现登录页。
- token 保存、退出登录、未登录跳转。
- 实现后台 `Layout`、侧边菜单、顶部栏、403/404。
- 菜单从 `GET /api/v1/admin/menus` 获取或用种子权限兜底。

## 不要做

- 不复刻旧系统 Logo、配色、文案和图片。
- 不在页面里散落拼接 API URL。

## 验收

```bash
cd admin-web
npm run lint
npm run build
```

手工验收：

- 未登录访问后台业务页跳转登录。
- 登录后显示菜单和基础 Layout。
