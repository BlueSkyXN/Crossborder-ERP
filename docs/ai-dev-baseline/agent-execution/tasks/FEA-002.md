# FEA-002 后台基础配置页面

phase: `P1_IDENTITY_CONFIG`  
depends_on: `BE-004`, `FEA-001`  
next: `FEU-001A`

## 目标

实现仓库、渠道、包装、增值服务、费率简版管理页面。

## 必读

- `../../15-frontend-style-from-screenshots-and-repro.md`
- `../../14-module-implementation-spec.md`
- `../../04-api-database-contract.md`

## 必须做

- 仓库列表、新增、编辑、启停。
- 发货渠道列表、新增、编辑、启停。
- 包装方式和增值服务管理。
- 简版费率管理。
- 列表要有筛选、分页、loading、empty、error。
- 高风险操作二次确认。

## 不要做

- 不做复杂价格引擎 UI。
- 不做旧系统视觉复刻。

## 验收

```bash
cd admin-web
npm run lint
npm run build
```

联动验收：

- 后台新增仓库后，用户端仓库地址接口可见。
