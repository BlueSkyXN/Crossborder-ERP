# FEA-003 后台待入库、扫描入库、包裹详情

phase: `P2_PARCEL_WMS`  
depends_on: `BE-005`, `FEA-001`  
next: `FEU-001B`

## 目标

实现后台处理包裹预报和仓库入库的高频页面。

## 必读

- `../../15-frontend-style-from-screenshots-and-repro.md`
- `../../14-module-implementation-spec.md`
- `../../04-api-database-contract.md`

## 必须做

- 待入库包裹列表。
- 扫描入库页：大输入框、搜索、重量/体积/图片/备注。
- 包裹详情页或抽屉。
- 在库包裹列表。
- 无主包裹登记简版。
- loading、empty、error、状态标签。

## 不要做

- 不做 PDA/打印机深度集成。
- 不做复杂批量导入。

## 验收

```bash
cd admin-web
npm run lint
npm run build
```

联动验收：

- 用户提交预报后，后台能搜索并入库。
- 入库后用户端看到 `IN_STOCK`。
