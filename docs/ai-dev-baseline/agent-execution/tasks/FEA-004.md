# FEA-004 后台运单、费用、充值、发货、轨迹

phase: `P3_WAYBILL_FINANCE`  
depends_on: `BE-006`, `BE-007`, `BE-008`, `FEA-001`  
next: `FEU-001C`

## 目标

实现后台运单审核、设置费用、人工充值、发货和轨迹录入。

## 必读

- `../../15-frontend-style-from-screenshots-and-repro.md`
- `../../02-domain-model-state-machines.md`
- `../../04-api-database-contract.md`

## 必须做

- 运单列表和状态 Tab。
- 运单详情：包裹、用户、地址、费用、轨迹、操作日志。
- 审核和设置费用。
- 人工充值入口。
- 发货和添加轨迹。
- 状态按钮按状态机展示。

## 不要做

- 不做复杂批次发货和面单打印。
- 不绕过 API 直接改状态。

## 验收

```bash
cd admin-web
npm run lint
npm run build
```

联动验收：

- 后台能把 Waybill 推进到 `PENDING_PAYMENT`。
- 充值后用户余额变化。
- 发货后用户端可查轨迹。
