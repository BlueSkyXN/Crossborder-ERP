# FIN-001 线下汇款与财务中心

phase: `P6_PRODUCTION_GAP`
depends_on: `FILE-001`
next: `MSG-001`

## 目标

补齐用户线下汇款提交、凭证上传、后台审核入账、用户财务流水查看能力。当前继续遵守 no-Docker、SQLite-first，不接真实支付网关。

## 来源

- ChatGPT/Gemini User Web 均要求线下汇款、凭证上传、汇款记录和财务流水。
- ChatGPT/Gemini Admin 均要求线下汇款审核和财务入账。
- `FILE-001` 已提供本地文件上传和凭证 file id 基础。

## 必须做

- 新增或扩展后端 finance 汇款单模型、状态机、serializer、service、API、migration 和测试。
- 用户可提交线下汇款单，必须引用 `REMITTANCE_PROOF` 文件。
- 后台财务可审核通过/取消；通过后给钱包入账并生成流水，重复审核必须防重。
- User Web/Mobile H5 至少提供提交入口或财务记录入口；Admin Web 提供审核入口。
- 更新 README/部署边界/Agent run 记录。

## 不要做

- 不接真实网银、Stripe、PayPal、支付宝、微信支付或支付回调。
- 不把线下汇款说成线上支付。
- 不引入 Docker/PostgreSQL/MySQL/Redis。

## 验收

```bash
cd backend && uv run python manage.py makemigrations --check --dry-run
cd backend && uv run pytest
pnpm lint
pnpm build
npm run e2e
git diff --check
```

补充验收：

- 用户只能看自己的汇款单。
- 财务审核通过后钱包余额和流水一致。
- 已审核汇款单不能重复入账。
