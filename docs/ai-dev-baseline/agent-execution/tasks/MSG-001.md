# MSG-001 客服消息与工单入口

phase: `P6_PRODUCTION_GAP`
depends_on: `FIN-001`
next: `MEMBER-001`

## 目标

补齐用户留言、附件、后台客服回复/处理和用户查看状态能力。当前继续遵守 no-Docker、SQLite-first，不接第三方客服系统。

## 来源

- ChatGPT/Gemini Admin、User Web、Mobile 均要求消息、留言、客服或工单入口。
- `FILE-001` 已提供 `MESSAGE_ATTACHMENT` 上传能力，可作为附件基础。

## 必须做

- 新增或实现 `tickets`/message 后端模型、状态机、serializer、service、API、migration 和测试。
- 用户可创建工单/留言，必要时可引用 `MESSAGE_ATTACHMENT` 文件。
- 后台客服可查看、回复、关闭或标记处理中。
- 用户只能查看自己的消息/工单；后台接口走 RBAC。
- Admin Web 提供客服处理入口；User Web/Mobile H5 至少提供消息入口、创建和查看记录。
- 更新 README/部署边界/Agent run 记录。

## 不要做

- 不接真实在线客服、短信、邮件、WhatsApp、微信或第三方 IM。
- 不引入 Redis/Celery 作为消息实时推送前提。
- 不做复杂客服 SLA、自动分派或机器人回复。

## 验收

```bash
cd backend && uv run python manage.py makemigrations --check --dry-run
cd backend && uv run pytest
pnpm lint
pnpm build
npm run e2e
git diff --check
```

补充验收：

- 用户只能看自己的工单和消息。
- 后台客服回复后用户端可读取。
- 附件必须校验存在、归属和 `MESSAGE_ATTACHMENT` 用途。
