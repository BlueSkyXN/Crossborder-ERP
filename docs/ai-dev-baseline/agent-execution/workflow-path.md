# 任务路径总图

本文件是人工和 Agent 都能读的任务路径。机器可读版本见 `task-graph.yaml`。

## P0_FOUNDATION：工程基础

```text
01 INIT-001 创建 monorepo
02 BE-001 初始化 Django/DRF 后端
```

完成标志：

- 后端基础接口可用。
- PostgreSQL/Redis 可启动。
- 三端前端项目可 build 或至少已初始化。

## P1_IDENTITY_CONFIG：账号、权限、仓库配置

```text
03 BE-002 后台 IAM/RBAC
04 BE-003 用户和会员资料
05 BE-004 仓库和基础配置
06 FEA-001 后台登录和 Layout
07 FEA-002 后台基础配置页面
08 FEU-001A 用户 Web 登录、控制台、仓库地址
09 FEM-001A 移动 H5 登录、底部导航、寄件首页
```

完成标志：

- 管理员能登录后台。
- 用户能登录用户 Web / mobile H5。
- 后台配置仓库后，用户端能看到专属仓库地址。

## P2_PARCEL_WMS：包裹预报和仓储入库

```text
10 BE-005 包裹预报和扫描入库
11 FEA-003 后台待入库、扫描入库、包裹详情
12 FEU-001B 用户 Web 包裹预报、包裹列表、包裹详情
13 FEM-001B 移动 H5 发布预报、包裹详情、申请打包入口
```

完成标志：

- 用户提交 `tracking_no = TEST123`。
- 后台能搜索并入库。
- 用户端看到 Parcel 变为 `IN_STOCK`。

## P3_WAYBILL_FINANCE：运单、费用、钱包支付、轨迹

```text
14 BE-006 运单和打包
15 BE-007 钱包和余额支付
16 BE-008 物流轨迹
17 FEA-004 后台运单、费用、充值、发货、轨迹
18 FEU-001C 用户 Web 申请打包、运单、支付、轨迹
19 FEM-001C 移动 H5 运单列表、追踪、确认收货
```

完成标志：

- `Parcel IN_STOCK -> Waybill PENDING_REVIEW`。
- 后台审核运单并设置费用。
- 后台充值后，用户余额支付。
- 后台发货并添加轨迹。
- 用户确认签收。

## P4_PURCHASE：最小代购链路

```text
20 BE-009 最小代购和到货转 Parcel
21 FEA-005 后台代购处理页面
22 FEU-002 用户 Web 手工代购、商品、购物车、确认订单
23 FEM-002 移动 H5 商品、购物车、手工代购入口
```

完成标志：

- 用户提交手工代购。
- 后台审核、采购、到货。
- 到货转成 Parcel。
- Parcel 可以继续走申请打包发货。

## P5_DELIVERY：端到端验收和交付

```text
24 E2E-001 端到端验收脚本
25 DOC-001 交付文档、部署说明、演示脚本
```

完成标志：

- 新机器按 README 能启动。
- E2E 或手工脚本可复现 P0 主链路。
- 已知问题和下一阶段计划清楚。
