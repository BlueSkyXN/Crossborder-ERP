# 01 轻量 MVP 功能需求规格

## 1. 用户端需求

### 1.1 首页

目标：用户进入后能立刻理解平台能做什么。

首页区块：

1. Hero 区：
   - 标题：跨境代购与集运，一站式处理。
   - 主按钮：立即预报包裹。
   - 次按钮：提交代购需求。
2. 流程说明：
   - 获取仓库地址。
   - 国内下单寄仓库。
   - 包裹入库。
   - 合箱付款。
   - 国际发货。
3. 常用入口：
   - 仓库地址。
   - 包裹预报。
   - 我的包裹。
   - 运费估算。
   - 财务充值。
4. 公告和帮助：
   - 最新公告。
   - 禁运说明。
   - 费用说明。

验收：

- 未登录用户可看到首页、商品/帮助/运费估算。
- 点击需要登录的入口时跳转登录页。

### 1.2 登录、注册、账户

功能：

- 邮箱登录。
- 邮箱注册。
- 找回密码。
- 修改密码。
- 账户资料。
- 会员仓库识别码展示。

P0 不做：

- 短信验证码。
- 微信登录。
- MFA。

### 1.3 仓库地址

功能：

- 用户查看平台国内仓库地址。
- 显示会员专属识别码。
- 一键复制完整收货地址。
- 说明用户下单时必须填写识别码。

字段：

- 仓库名称。
- 收货人。
- 电话。
- 省市区。
- 详细地址。
- 会员识别码。
- 完整地址。

验收：

- 用户登录后可复制专属地址。
- 地址中包含会员识别码。

### 1.4 包裹预报

功能：

- 选择仓库。
- 输入快递单号。
- 输入承运商。
- 填写商品名称、数量、申报价值。
- 提交预报。

P0 简化：

- 不强制上传图片。
- 不做复杂商品属性。
- 不做批量导入作为首要功能。

验收：

- 提交后包裹状态为 `PENDING_INBOUND`。
- 用户只能看到自己的包裹。

### 1.5 我的包裹

功能：

- 查看待入库、在库、已申请打包、已出库包裹。
- 搜索快递单号。
- 选择在库包裹申请打包。

验收：

- 只有 `IN_STOCK` 包裹可申请打包。
- 不同仓库包裹不能合并申请同一运单。

### 1.6 运单与打包

功能：

- 选择包裹。
- 选择发货渠道。
- 选择海外收件地址。
- 提交打包申请。
- 查看运单费用。
- 余额支付。
- 查看轨迹。
- 确认收货。

验收：

- 后台计费前不可支付。
- 余额不足时提示充值。
- 支付后进入待发货。
- 发货后可查看轨迹。

### 1.7 万能代购

功能：

- 粘贴商品链接。
- 填写商品名称。
- 填写规格、数量、预估价格。
- 提交代购需求。
- 查看代购订单状态。

P0 简化：

- 不真实抓取淘宝/天猫页面。
- 不自动下单。
- 不自动同步外部订单。

验收：

- 后台审核后可进入采购状态。
- 到货后可转为用户在库包裹。

### 1.8 财务中心

功能：

- 查看余额。
- 查看流水。
- 提交线下汇款。
- 查看汇款审核状态。

P0 支付能力：

- 后台人工充值。
- 用户线下汇款，后台审核入账。
- 余额支付运费。

P0 不做：

- 微信支付。
- 支付宝。
- PayPal。
- Qpay。
- 信用卡。

这些只在 UI 中作为 `后续接入` 或后台配置占位，不作为已完成能力。

### 1.9 工单与消息

功能：

- 用户提交咨询。
- 后台回复。
- 用户查看回复。

P0 简化：

- 不做实时聊天。
- 不做客服分配算法。

### 1.10 无主包裹认领

P1 功能，P0 可保留已有能力。

功能：

- 用户输入或搜索快递单号。
- 系统脱敏展示匹配结果。
- 用户提交认领说明。
- 后台审核通过后转为用户包裹。

## 2. 后台需求

### 2.1 控制台

展示：

- 今日入库数。
- 待审核运单。
- 待付款运单。
- 待发货运单。
- 待处理代购。
- 待审核汇款。
- 近 7 日订单/运单趋势。

P0 不做复杂 BI。

### 2.2 包裹入库

功能：

- 扫描或输入快递单号。
- 匹配用户预报。
- 录入重量、尺寸、备注。
- 入库后状态变为 `IN_STOCK`。
- 未匹配时生成无主包裹。

P0 简化：

- 不接扫码枪驱动；扫码枪作为键盘输入即可。
- 不强制库位。

### 2.3 运单处理

后台分组：

- 待审核。
- 待打包/待计费。
- 待付款。
- 待发货。
- 已发货。
- 问题单。

操作：

- 审核通过。
- 设置费用。
- 取消。
- 发货。
- 追加轨迹。

### 2.4 发货批次

P1 功能，已有基础可保留。

功能：

- 创建批次。
- 选择待发货运单。
- 锁定批次。
- 批量发货。
- 批量追加轨迹。
- 生成面单/拣货单/交接单预览数据。

P0 不接真实打印机。

### 2.5 代购处理

功能：

- 查看待审核代购。
- 审核通过。
- 标记已采购。
- 填写采购成本、外部订单号、国内快递单号。
- 标记到货。
- 转为用户在库包裹。

### 2.6 财务管理

功能：

- 用户余额查看。
- 后台人工充值。
- 后台人工扣减。
- 线下汇款审核通过/驳回。
- 查看流水。
- 查看应付基础。

P0 不做：

- 真实银行付款。
- 自动对账。
- 复杂总账。

### 2.7 商品管理

功能：

- 商品分类。
- 商品。
- SKU。
- 价格。
- 库存。
- 上架/下架。

用途：

- 平台自营商品。
- 常用代购商品展示。
- 后续可接购物车代购。

### 2.8 会员管理

功能：

- 会员列表。
- 搜索。
- 冻结/解冻。
- 重置密码。
- 会员等级。
- 客服备注。

### 2.9 内容管理

功能：

- 公告。
- 帮助中心。
- 禁运说明。
- 运费说明。
- 关于我们。
- 条款/隐私。

### 2.10 基础设置

功能：

- 仓库。
- 国家/地区。
- 发货渠道。
- 包装方式。
- 增值服务。
- 运费估算规则。

P0 运费规则只做首重/续重/体积重基础版本。

## 3. 推广与积分

P1 功能。

轻量实现：

- 邀请码。
- 邀请人数统计。
- 后台手工登记返利。
- 积分流水。
- 不做提现。
- 不做多级分销。

## 4. 核心验收剧本

### 剧本 A：集运闭环

```text
用户注册/登录
-> 查看仓库地址
-> 提交包裹预报
-> 后台扫描入库
-> 用户申请打包
-> 后台审核并设置费用
-> 用户提交线下汇款
-> 后台审核入账
-> 用户余额支付运费
-> 后台发货并添加轨迹
-> 用户查看轨迹并确认收货
```

### 剧本 B：代购闭环

```text
用户提交万能代购链接
-> 后台审核
-> 后台标记已采购
-> 后台标记到货并转包裹
-> 用户对代购包裹申请打包
-> 进入集运闭环
```

### 剧本 C：后台管理闭环

```text
后台登录
-> 查看控制台待办
-> 处理入库
-> 处理运单
-> 审核汇款
-> 发货
-> 查看审计日志
```
